# Free Lines | النظام المالي — تصميم الواجهة (Static UI)

واجهة React كاملة لنظام إدارة العهد والمصروفات، مبنية على تصميم الموك أب المرفق.
البيانات كلها Mock (ثابتة) حالياً — لسه مش متربطة بـ Firebase، عشان نراجع الشكل والتنقل الأول.

## تشغيل المشروع محلياً

```bash
npm install
npm run dev
```

هيفتح على `http://localhost:5173`

## البناء للنشر (Build)

```bash
npm run build
```

الملفات الناتجة هتبقى في مجلد `dist/` — قابلة للرفع على أي استضافة static
(GitHub Pages, Netlify, Firebase Hosting...).

## هيكل المشروع

```
src/
  data/mockData.js       بيانات وهمية (موظفين، مشاريع، مصروفات، عهد، فواتير، إشعارات)
  utils/colors.js        ربط أسماء الألوان بكلاسات Tailwind
  components/            عناصر مشتركة (Card, StatCard, WalletCard, IconBadge...)
  layouts/
    EmployeeShell.jsx     الهيكل العام لشاشات الموظف (Bottom nav + Top bar)
    AdminShell.jsx        الهيكل العام لشاشات الأدمن (Sidebar + Header)
  pages/
    Login.jsx              شاشة تسجيل الدخول (فيها اختيار موظف/مدير للمعاينة)
    employee/               كل شاشات الموظف (الرئيسية، المشاريع، إضافة مصروف، المصروفات، كشف الحساب، الإشعارات)
    admin/                  كل شاشات الأدمن (لوحة التحكم، الموظفون، المشاريع، العهد، المصروفات، الفواتير، التقارير، الإشعارات، الإعدادات)
```

## الخطوة الجاية

لما تحب نربطها بـ Firebase الموجود عندك في مشروع النظام المالي،
هنستبدل `src/data/mockData.js` بقراءة حقيقية من Firestore، ونفعّل تسجيل الدخول
الحقيقي بدل الاختيار اليدوي (موظف/مدير) اللي في شاشة الدخول.
